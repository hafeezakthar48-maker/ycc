# China Finance AI OS 产品需求文档（PRD）

## 第二章：功能需求说明（FRD V1.0）

版本：V1.0

---

## 一、总体功能架构

系统由十二个一级模块组成：

```text
China Finance AI OS
├── AI首页（Dashboard）
├── AI财务中心
├── AI税务中心
├── AI经营分析中心
├── BI数据中心
├── 电商分析中心
├── OCR识别中心
├── 企业知识库
├── AI智能助手
├── 风险预警中心
├── 系统管理
└── 开放平台（API）
```

所有模块支持插件化，可独立升级，不影响其他模块。

## 三、AI财务中心

### 功能目标

替代大量重复财务工作。

### 1. 智能记账

支持：

- 自动识别业务
- 自动推荐会计科目
- 自动生成凭证
- 自动审核凭证

### 2. 凭证中心

支持：

- 新增
- 修改
- 审核
- 反审核
- 导入
- 导出
- 自动编号
- 附件上传
- OCR识别附件
- AI检查凭证错误

### 3. 总账

支持：

- 总账
- 明细账
- 科目余额表
- 辅助核算
- 多账套
- 多币种

当前正式核算接入状态：

- 已建设正式核算一期底座，凭证中心已审核凭证可生成不可变正式分录。
- 总账、明细账、科目余额表和财务报表优先读取正式分录来源。
- 反过账采用冲销分录，保留来源凭证、操作人、账套、期间和审计日志。
- 后续阶段继续建设辅助核算、期末结转、电子凭证归档、期末外币重估和完整多账套治理。

当前多币种接入状态：

- 已支持外币正式分录和汇率表。
- 已支持原币明细账展示和本位币总账汇总。
- 已支持财务报表按账套本位币生成，并提示外币分录数量。
- 期末外币重估、汇兑损益和多币种合并报表放入后续期末处理阶段。

当前辅助核算接入状态：

- 已支持客户、供应商、员工、部门、项目、资产、平台和 SKU 维度主数据。
- 已支持正式分录行挂载辅助核算维度。
- 已支持明细账按辅助核算维度过滤。
- 应收应付账龄、项目成本、固定资产卡片、部门预算和 SKU 利润分析进入后续专业子模块。

### 4. 固定资产

支持：

- 新增
- 折旧
- 报废
- 出售
- 盘点
- 自动折旧计算

当前 MVP 接入状态：

- 已落地固定资产台账、新增资产、直线法自动折旧、报废、出售、盘点、账套隔离、权限控制和审计日志。
- 暂不生成正式折旧凭证、总账分录、资产卡片附件、资产减值或税会差异调整。

### 5. 工资管理

支持：

- 工资计算
- 社保
- 公积金
- 个税计算
- 工资分析

当前 MVP 接入状态：

- 已落地工资计算、社保、公积金、个税、实发工资、企业成本、部门工资分析、权限控制和审计日志。
- 暂不处理累计预扣预缴、城市社保公积金差异、银行代发、工资条发送、个税正式申报或工资凭证自动生成。

### 6. 报表

自动生成：

- 资产负债表
- 利润表
- 现金流量表
- 所有者权益变动表
- 管理报表

当前 MVP 接入状态：

- 已落地资产负债表、利润表、现金流量表、所有者权益变动表和管理报表摘要自动生成。
- 支持优先基于正式分录取数；当账套无正式分录时，回退已审核凭证；当账套无已审核凭证时，回退样例经营数据生成演示报表。
- 已接入 `statement.generate` 权限控制和 `statement.generate` 审计日志。
- 暂不覆盖合并报表、复杂金融工具、长期股权投资、递延所得税、现金流量表补充资料、附注披露或正式申报报表。
