# China Finance AI OS 产品需求文档（PRD）

## 第二章：功能需求说明（FRD V1.0）

版本：V1.0

---

## 一、总体功能架构

系统由十二个一级模块组成：

```text
China Finance AI OS
├── AI首页（Dashboard）
├── AI财务中心
├── AI税务中心
├── AI经营分析中心
├── BI数据中心
├── 电商分析中心
├── OCR识别中心
├── 企业知识库
├── AI智能助手
├── 风险预警中心
├── 系统管理
└── 开放平台（API）
```

所有模块支持插件化，可独立升级，不影响其他模块。

## 二、AI首页（Dashboard）

### 功能目标

为企业管理层提供一屏式经营总览。

### 页面内容

经营概况：

- 今日销售额
- 本月销售额
- 本年销售额

利润：

- 毛利润
- 净利润
- 利润率

现金流：

- 流入
- 流出
- 余额

库存：

- 库存金额
- 库存周转率
- 滞销库存

税务：

- 本月应纳税额
- 税负率
- 风险数量

AI提示每日自动生成：

- 风险提示
- 异常分析
- 今日经营建议

## 三、AI财务中心

### 功能目标

替代大量重复财务工作。

### 1. 智能记账

支持：

- 自动识别业务
- 自动推荐会计科目
- 自动生成凭证
- 自动审核凭证

### 2. 凭证中心

支持：

- 新增
- 修改
- 审核
- 反审核
- 导入
- 导出
- 自动编号
- 附件上传
- OCR识别附件
- AI检查凭证错误

### 3. 总账

支持：

- 总账
- 明细账
- 科目余额表
- 辅助核算
- 多账套
- 多币种

当前正式核算接入状态：

- 已建设正式核算一期底座，凭证中心已审核凭证可生成不可变正式分录。
- 总账、明细账、科目余额表和财务报表优先读取正式分录来源。
- 反过账采用冲销分录，保留来源凭证、操作人、账套、期间和审计日志。
- 已接入多币种、辅助核算维度、期间结账四期、报表快照归档、电子会计档案 Phase 7、往来核算 Phase 8、银行对账 Phase 9、固定资产正式核算 Phase 10、薪酬正式核算 Phase 11、存货与成本核算 Phase 12、税务核算 Phase 13、预提摊销与融资利息 Phase 14 和合并报表 Phase 15；后续阶段继续建设完整多账套治理和专业子模块。

当前多币种接入状态：

- 已支持外币正式分录和汇率表。
- 已支持原币明细账展示和本位币总账汇总。
- 已支持财务报表按账套本位币生成，并提示外币分录数量。
- 已支持期间结账中的期末外币重估和汇兑损益分录；多币种合并报表仍放入后续阶段。

当前辅助核算接入状态：

- 已支持客户、供应商、员工、部门、项目、资产、平台、SKU 和仓库维度主数据。
- 已支持正式分录行挂载辅助核算维度。
- 已支持明细账按辅助核算维度过滤。
- 项目成本、固定资产卡片附件、部门预算、SKU 利润分析、合同/订单驱动的完整往来台账和复杂重分类报表进入后续专业子模块。

当前期间结账接入状态：

- 已落地结账运行、结账检查清单、自动期末分录、期间关闭/重开、权限控制和审计日志。
- 已支持固定资产折旧、工资计提、税费计提、附加税计提、预提摊销、外币期末重估、坏账准备、存货成本结转汇总、月末损益结转和年终利润分配动作。
- 外币重估按期末汇率只生成本位币调整分录，不改变原币余额；损益结转将当期收入、成本和费用归集至 `4103 本年利润`。
- 关闭期间后拒绝新增正式分录、凭证过账和重新生成期末动作；重开期间保留既有正式分录与审计轨迹。
- 当前仍不覆盖完整纳税申报、银行代发、真实网银直连、银企直连签名、电子会计档案国家标准全量字段、CA 签章、长期冷备、生产制造复杂成本结转、复杂审计合并、跨账套自动抵销、供应链订单、合同台账、自动付款或完整催收流程。

### 3.1 电子会计档案（Phase 7）

支持：

- 凭证附件上传后生成档案文档索引，并关联来源凭证、账套和会计期间。
- 计算上传字节的 SHA-256，记录文件名、content type、大小、存储状态、OCR 状态、验真状态和保管期限。
- 文本附件可保存文本摘录；图片或 PDF 在未接入真实 OCR 引擎时标记为 `ocr_status="engine_required"`。
- 发票和银行回单在未接入外部验真服务前标记为 `verification_status="pending_external"`。
- 支持按账套、期间和案卷类型创建会计档案案卷，并下载包含 `manifest.json`、文档元数据 JSON 和文本摘录的 ZIP 归档包。
- 已接入 `archive.read`、`archive.document.create`、`archive.case.create`、`archive.package.download`、`archive.verification.update` 权限控制，以及档案文档查询、案卷创建和归档包下载审计事件。

当前边界：

- 不把原始二进制永久写入 SQLite，当前默认 `storage_status="metadata_only"`。
- 不提供 CA 签章、官方电子发票实时验真、对象存储 WORM、长期冷备、电子会计档案国标全量字段、跨系统档案移交或档案销毁审批。

### 3.2 往来核算（Phase 8）

支持：

- 基于正式已过账分录行生成应收、应付未清项和往来对象余额。
- 应收识别 `1122/1221` 且要求客户维度；应付识别 `2202/2241` 且要求供应商维度。
- 支持按期间月末或指定截止日输出账龄分析，账龄桶为 `0-30`、`31-60`、`61-90`、`91-180`、`181-365` 和 `365+`。
- 支持同一往来对象内的部分核销，核销只追加匹配记录，不改写历史正式分录。
- 支持期间结账动作 `bad_debt_provision`，按账龄规则生成借 `6701`、贷 `1231` 的坏账准备分录。
- 已接入 `receivable_payable.read`、`receivable_payable.settle`、`receivable_payable.bad_debt` 权限控制，以及余额查询、账龄查询、核销和坏账准备审计事件。

当前边界：

- 不从凭证摘要、报表快照或样例数据反推往来余额。
- 不覆盖销售订单、采购订单、合同台账、银行流水自动抓取、自动付款、信用额度审批、完整催收流程、预收预付复杂重分类或复杂坏账组合模型。

### 3.3 银行对账（Phase 9）

支持：

- 导入银行流水并按账套、银行账户和银行流水号去重。
- 基于正式已过账资金分录生成匹配候选，资金科目范围为 `1001/1002/1012`。
- 按金额、日期和摘要生成匹配评分，并保留匹配依据。
- 支持人工确认银行流水与正式资金分录的对账关系，确认记录只追加，不改写银行流水金额或正式分录。
- 支持银行余额调节表，展示银行账面余额、企业账面余额、四类未达账项和调节后余额。
- 支持确认对账时透传应收应付核销请求，联动往来核销服务。
- 已接入 `bank_reconciliation.read`、`bank_reconciliation.import`、`bank_reconciliation.match`、`bank_reconciliation.confirm` 权限控制，以及流水导入、候选生成、确认对账和调节表读取审计事件。

当前边界：

- 银行流水为手工/API 导入事实，不接真实网银，不保存网银登录凭据。
- 不做银企直连签名、自动付款、银行回单官方验真、多币种换汇拆分对账或完整出纳复核流程。

### 3.4 固定资产正式核算（Phase 10）

支持：

- 基于固定资产台账资产生成资本化正式分录，借记 `1601 固定资产`，贷记应付、银行或指定过渡科目。
- 基于已资本化资产逐资产生成正式折旧分录，借记 `6602 管理费用`，贷记 `1602 累计折旧`。
- 支持固定资产减值准备，借记 `6701 资产减值损失`，贷记 `1603 固定资产减值准备`。
- 支持通过 `1606 固定资产清理` 完成正式处置，结转原值、累计折旧、减值准备、处置收入和处置收益或损失。
- 支持正式资产卡片追溯，展示资本化分录、最近折旧分录、减值准备、处置分录、正式净值和正式核算状态。
- 已接入 `fixed_asset_accounting.read`、`fixed_asset_accounting.post`、`fixed_asset_accounting.impair`、`fixed_asset_accounting.dispose` 权限控制，以及卡片读取、资本化、折旧、减值和处置审计事件。

当前边界：

- 固定资产生命周期台账仍为内存 MVP，服务重启后演示资产会重置；正式分录保存在正式核算库。
- 正式核算通过 `asset` 辅助核算维度追溯资产，不从凭证摘要、报表缓存或样例数据反推固定资产卡片。
- 已关闭期间拒绝新增固定资产正式资本化、折旧、减值和处置分录；同一来源键重复调用返回既有正式分录。
- 不提供固定资产卡片附件、复杂融资租赁、资产评估接口、税会差异自动申报、集团跨法人调拨或完整资产清查审批流程。

### 3.5 存货与成本核算（Phase 12）

支持：

- 按账套、SKU 和仓库维护库存余额，展示库存数量、金额和移动加权平均成本。
- 采购入库生成正式分录，借记 `1405 库存商品`，贷记 `2202 应付账款`，并更新移动平均成本。
- 销售出库按移动平均成本结转主营业务成本，借记 `6401 主营业务成本`，贷记 `1405 库存商品`。
- 库存不足时拒绝销售出库，避免正式成本结转导致数量或金额为负。
- 存货跌价准备生成借记 `6701 资产减值损失`、贷记 `1471 存货跌价准备` 的正式分录。
- 盘点盘盈盘亏先进入 `1901 待处理财产损溢`，并保留审批人和审批时间。
- 期间结账支持 `inventory_cost_rollforward` 动作，用于汇总校验当期销售出库成本分录。
- 前端 AI 财务中心展示 SKU 库存余额、入库出库流水、销售成本结转金额、跌价和盘点差异状态。
- 已接入 `inventory_accounting.read`、`inventory_accounting.receipt`、`inventory_accounting.issue`、`inventory_accounting.impair`、`inventory_accounting.count` 权限控制，以及余额读取、采购入库、销售出库、跌价准备和盘点差异审计事件。

当前边界：

- 库存台账为内存 MVP，正式金额事实保存在正式分录库。
- 正式核算通过 `sku` 和 `warehouse` 辅助核算维度追溯库存地点和商品。
- 已关闭期间拒绝新增采购入库、销售出库、跌价准备和盘点差异正式分录。
- 不接 WMS 实时库存、不管理批次保质期、不做生产制造 BOM 或工序成本、不从跨境电商平台真实 API 抓取订单。

### 3.6 税务核算与申报底稿（Phase 13）

支持：

- 从正式分录抽取 `22210101` 进项税额、`22210102` 销项税额和 `22210104` 进项税额转出，形成增值税台账。
- 申报底稿展示销项税额、进项税额、进项转出、应交增值税、附加税和企业所得税，并保留来源分录追溯。
- 未交增值税结转生成借 `22210103`、贷 `222102` 的正式分录。
- 附加税计提生成借 `6403`、贷 `222103` 的正式分录，并接入期间结账动作 `tax_surtax_accrual`。
- 企业所得税计提生成借 `6801`、贷 `222104` 的正式分录。
- 纳税支付生成借税费明细科目、贷 `1002 银行存款` 的正式分录。
- 前端 AI 财务中心展示增值税台账、申报底稿、计提状态和缴款入口。
- 已接入 `tax_accounting.read`、`tax_accounting.accrue`、`tax_accounting.pay` 权限控制，以及台账读取、底稿读取、税费结转、税费计提和缴款审计事件。

当前边界：

- 申报底稿只作为计算和复核依据，不自动提交正式纳税申报。
- 不接税局直连接口，不自动做抵扣认证，不做真实发票验真。
- 不处理复杂税收优惠备案、递延所得税、跨地区复杂税种分摊或税务师专业判断。
- 已关闭期间拒绝新增税费计提、结转和缴款正式分录；同一来源键重复调用返回既有正式分录。

### 3.7 预提摊销与融资利息（Phase 14）

支持：

- 创建预付摊销、预提费用、递延收入和借款利息计划，记录账套、计划编号、起止期间、总金额、科目和已生成期间。
- 按月平均分摊计划金额，最后一期吸收尾差，确保累计入账金额等于计划总额。
- 生成计划本期正式分录，来源键为 `schedule_posting:{account_set_id}:{period}:{schedule_code}`，重复调用返回既有分录。
- 生成借款利息计提正式分录，按本金和年利率计算月息，来源键为 `loan_interest_accrual:{account_set_id}:{period}:{loan_code}`。
- 期间结账支持 `accrual_amortization_posting` 动作，按活跃核算计划生成本期幂等分录。
- 前端 AI 财务中心展示核算计划、借款计划、本期金额、已生成期间，并提供创建计划、生成本期和计提借款利息入口。
- 已接入 `accrual_amortization.read`、`accrual_amortization.write`、`accrual_amortization.post` 权限控制，以及计划读取、计划创建、计划过账和借款利息计提审计事件。

当前边界：

- 计划表为内存 MVP，用于验证正式分录、期间结账和前端流程；服务重启后演示计划会重置。
- 不做复杂金融工具公允价值、实际利率摊余成本完整模型、租赁准则全流程或合同收入五步法自动判断。
- 已关闭期间拒绝新增预提摊销和借款利息正式分录；同一来源键重复调用返回既有正式分录。

### 3.8 合并报表与内部抵销（Phase 15）

支持：

- 维护合并集团、纳入账套、主体名称、持股比例和合并方法。
- 读取单体账套资产负债表、利润表和现金流量表报表包。
- 生成内部应收应付抵销，借应付、贷应收，抵销分录只存在于合并层。
- 生成内部销售收入与成本抵销，以及期末存货未实现内部利润抵销。
- 生成母公司长期股权投资与子公司权益抵销，并计算少数股东权益和少数股东损益。
- 前端 AI 财务中心展示合并范围、单体报表包状态、抵销工作底稿、合并报表摘要和少数股东权益。
- 已接入 `consolidation.read`、`consolidation.write`、`consolidation.rebuild` 权限控制，以及合并范围读取/维护、报表包读取、抵销重建和合并报表读取审计事件。

当前边界：

- 合并范围和抵销底稿为内存 MVP，用于验证多账套合并流程；服务重启后演示集团和抵销底稿会重置。
- 抵销分录不回写单体账套正式分录，单体税务、期间关闭和正式分录审计轨迹保持独立。
- 不做复杂股权购买法追溯、购买日公允价值分摊、商誉减值完整评估、境外准则转换、复杂交叉持股或审计合并底稿替代。

### 4. 固定资产

支持：

- 新增
- 折旧
- 报废
- 出售
- 盘点
- 自动折旧计算
- 正式资本化
- 正式折旧
- 减值准备
- 清理处置
- 正式资产卡片追溯

当前接入状态：

- 已落地固定资产台账、新增资产、直线法自动折旧、报废、出售、盘点、账套隔离、权限控制和审计日志。
- 已落地固定资产正式核算 Phase 10，支持资本化、逐资产正式折旧、减值准备、清理处置、处置损益和正式卡片追溯。
- 暂不提供固定资产卡片附件、复杂融资租赁、资产评估接口、税会差异自动申报、集团跨法人调拨或完整资产清查审批流程。

### 5. 工资管理

支持：

- 工资计算
- 社保
- 公积金
- 个税计算
- 工资分析
- 工资计提正式分录
- 工资发放正式分录
- 个税、社保和公积金缴纳正式分录

当前 MVP 接入状态：

- 已落地工资计算、社保、公积金、个税、实发工资、企业成本、部门工资分析、权限控制和审计日志。
- 已落地薪酬正式核算 Phase 11，支持默认工资批次、工资和企业社保公积金计提、工资发放、个税与社保公积金缴纳、正式分录来源键、权限控制和审计日志。
- 已关闭期间拒绝新增工资计提、发放和缴纳分录；工资发放前必须存在计提分录。

## Phase 16：正式核算上线治理

正式核算上线治理作为 AI 财务中心的上线控制层，不新增业务核算口径，只检查和保护已有正式分录、账簿、报表、归档和权限配置。

功能范围：

- 完整性校验覆盖借贷平衡、来源键重复、期间锁定、报表映射和归档链接。
- MVP 凭证迁移支持 dry-run、差异报告和受控 apply。
- 备份清单覆盖正式核算核心数据集，并支持恢复演练结果记录。
- 权限矩阵覆盖上线所需权限和职责分离规则。
- 上线门禁聚合完整性、迁移、备份恢复、权限矩阵和回归命令状态。

风险控制：

- 不删除 MVP 历史数据。
- 不在没有备份和恢复演练的情况下执行迁移。
- 不允许同一人完成迁移执行和上线审批。
- 所有迁移、备份、恢复演练和门禁读取必须写入审计日志。
