# MVP 设计

## 主流程

```text
内置示例财务数据
  -> FastAPI 计算指标与风险
  -> React/ECharts 展示经营驾驶舱
  -> 模板规则生成 AI 经营报告草稿
```

## OCR 发票识别边界

- 支持粘贴 OCR 文本、手动录入发票文本、上传 `.txt` 文件。
- 支持提取发票类型、发票代码、发票号码、开票日期、购销方名称与税号、金额、税额、价税合计。
- 支持基础风险：发票号码缺失、销售方纳税人识别号缺失、金额与税额和价税合计不一致。
- 图片 / PDF 上传在当前 MVP 中只做入口和边界提示；未配置 OCR 引擎时返回 `engine_status=missing`，不生成伪识别结果。
- 正式入账、抵扣或报销仍需财务人员复核发票原件、业务合同、订单、验收、物流和资金流水。

## AI 凭证草稿边界

- 支持三类高频场景：费用采购、库存采购、销售收入。
- 支持按结算状态切换往来科目或银行存款科目。
- 支持输出借贷方向、科目编码、科目名称、金额、说明、借方合计、贷方合计和平衡状态。
- 支持基础风险：价税合计与金额税额不一致、借贷不平衡。
- 不自动生成正式凭证号，不写入总账或明细账，不替代制单、审核、复核、记账职责。

## 凭证中心 MVP 边界

- 支持新增、修改、审核、反审核和自动编号，编号规则为 `记-YYYYMM-0001`。
- 支持 JSON 批量导入、CSV 导出和附件上传记录。
- 凭证附件上传后会生成会计档案文档索引，记录 `archive_document_id`、`sha256_hash`、`storage_status` 和 OCR 状态。
- 支持每次新增或修改后调用 AI 自动审核规则，返回评分、评级、风险发现和整改建议。
- 当前凭证中心使用本地 SQLite 保存演示数据、审核状态、过账状态、附件元数据和月度编号序列，后端服务重启后仍可读取。
- 当前已提供默认账套、跨境电商账套、按账套隔离的会计期间关闭/重开状态、正式分录一期、总账、明细账、科目余额表只读视图和电子会计档案文档索引，并接入 `ledger.read`、`ledger.period.manage`、`accounting.account.read`、`accounting.entry.read` 权限控制和审计日志；过账会生成正式不可变分录，反过账会生成冲销分录，仍不执行完整期末结账、完整反结账、完整多账套核算、永久附件原件存储或完整数据库事务编排。
- 凭证审核状态用于产品流程演示，不替代企业内部正式审核制度。

## 正式会计核算引擎一期边界

- 支持已审核凭证正式过账为 `journal_entry` / `journal_line`。
- 支持正式分录借贷平衡、科目启用和期间状态校验。
- 支持反过账生成冲销分录，不删除原正式分录。
- 支持总账、明细账、科目余额表和财务报表优先读取正式分录。
- 当前不实现完整外币期末重估、完整辅助核算、损益结转、现金流量表自动拆分、电子会计档案国家标准全量字段或长期冷备。

## 多币种核算二期边界

- 支持 `CNY`、`USD`、`EUR`、`HKD` 币种清单。
- 支持按账套和日期维护外币兑本位币汇率。
- 支持正式分录行保存交易币种金额、汇率和本位币金额。
- 支持总账按本位币汇总，明细账展示原币金额、汇率和本位币金额。
- 财务报表仍按账套本位币生成。
- 当前不自动执行期末外币重估、汇兑损益结转或多本位币合并报表。

## 辅助核算维度三期边界

- 支持维护客户、供应商、员工、部门、项目、资产、平台和 SKU 辅助核算维度。
- 支持正式分录行挂载一个或多个辅助核算维度。
- 支持正式分录过账时校验维度主数据存在且启用。
- 支持明细账展示维度标签，并按维度过滤查询。
- 当前不实现完整应收应付账龄、项目成本归集、固定资产卡片联动、SKU 利润核算或部门预算控制。

## 期间结账引擎四期边界

- 支持按账套和期间发起结账运行，生成可追踪的 `run_id`、检查清单、动作结果和审计事件。
- 支持结账检查清单：期间存在、期间未关闭、正式分录借贷平衡、凭证已过账、科目有效、期末汇率准备、折旧/工资/税费准备状态。
- 支持自动生成固定资产折旧、工资计提、税费计提、外币期末重估、损益结转和年终利润分配分录。
- 外币期末重估按 `原币余额 × 期末汇率 - 账面本位币余额` 计算，只生成 CNY 本位币调整分录，不改变原币余额。
- 月结将收入、成本、费用类科目结转至 `4103 本年利润`；年结将 `4103 本年利润` 结转至 `4104 利润分配-未分配利润`。
- 同一账套、期间、动作和来源键重复执行时返回已有正式分录，避免重复过账。
- 期间关闭后拒绝新增正式分录、凭证过账和重新生成期末动作；重开期间只记录状态与审计事件，不删除已生成分录。
- 当前税费计提使用规则化输入，不替代正式纳税申报；不覆盖银行代发、电子会计档案国标全量字段、CA 签章、长期冷备、复杂成本结转、合并报表或跨账套抵销。

## AI 自动审核边界

- 支持审核凭证草稿和手动录入的凭证分录。
- 支持检查借贷平衡、价税勾稽、基础单据信息完整性、分录金额有效性、进项/销项税额方向。
- 支持输出评分、评级、检查项状态、错误清单、证据、整改建议和法规/准则引用。
- 当前为规则审核，不覆盖全部行业、全部科目、复杂合并报表、长期股权投资、金融工具等复杂会计判断。
- 审核结果默认需要人工复核，不构成最终审计意见。

## 固定资产台账 MVP 边界

- 支持新增固定资产，按账套记录资产名称、类别、购置日期、原值、残值、折旧月数、位置和保管人。
- 支持直线法自动计算月折旧额，并按会计期间幂等计提折旧，避免同一期间重复计提。
- 支持报废、出售和盘点操作，出售时按当前账面净值计算处置损益。
- 支持固定资产摘要统计：资产数量、在用数量、报废数量、出售数量、原值、累计折旧、账面净值和月折旧额。
- 当前固定资产台账使用内存数据结构保存演示资产，服务重启后会重置，不生成正式折旧凭证、总账分录、卡片附件或税会差异调整。
- 固定资产 API 接入 `fixed_asset.read`、`fixed_asset.write`、`fixed_asset.depreciate`、`fixed_asset.dispose`、`fixed_asset.inventory` 权限和 `fixed_asset.*` 审计日志。

## 工资管理 MVP 边界

- 支持录入员工、部门、基本工资、奖金、津贴、社保基数、公积金基数、专项附加扣除和其他扣款。
- 支持按简化参数计算员工社保、企业社保、员工公积金、企业公积金、应纳税所得额、个税、实发工资和企业用工成本。
- 支持汇总员工数、应发工资、社保、公积金、个税、实发工资、企业成本和平均实发工资。
- 支持按部门生成工资分析，展示部门人数、应发、实发和企业成本。
- 当前采用 MVP 简化口径：员工社保 10.5%、企业社保 26.3%、个人/企业公积金各 7%、基本扣除 5000 元和月度综合所得税率表。
- 当前不处理累计预扣预缴、城市差异、社保/公积金封顶基数、年终奖单独计税、银行代发、工资条发送、个税正式申报或工资凭证自动生成。
- 工资 API 接入 `payroll.calculate` 权限和 `payroll.calculate` 审计日志。

## 财务报表自动生成 MVP 边界

- 支持优先基于正式分录生成资产负债表、利润表、现金流量表、所有者权益变动表和管理报表摘要。
- 支持在当前账套无正式分录时回退已审核凭证；无已审核凭证时回退使用内置样例经营数据生成演示报表。
- 支持默认“中国企业会计准则通用报表映射”映射集，覆盖资产负债表、利润表、现金流量表和所有者权益变动表。
- 资产负债表按期末余额取数；利润表按期间发生额取数；现金流量表按 `cash_flow_item_code` 或现金科目/对方科目规则取数；所有者权益变动表按期初权益、净利润、利润分配和期末权益取数。
- 支持输出报表来源、已审核凭证数、资产负债平衡状态、净利率、资产负债率、现金流利润比、生成追溯 `trace_items` 和公式校验 `validation_items`。
- 支持创建报表快照版本，记录 `content_hash`、`validation_status`、`archive_status`、创建人和锁定人；锁定不会覆盖原快照。
- 支持从快照导出 Excel / PDF，Excel 包含四张标准报表、校验结果和追溯明细，PDF 当前为轻量归档摘要。
- 当前报表生成为单账套、单期间 MVP，不覆盖合并报表、复杂金融工具、长期股权投资、递延所得税、现金流量表补充资料、附注披露或正式申报报表。
- 财务报表 API 接入 `statement.generate`、`statement.validate`、`statement.mapping.view`、`statement.mapping.manage`、`statement.snapshot.create`、`statement.snapshot.lock`、`statement.archive.view`、`statement.export` 权限，并记录对应生成、映射、快照、锁定、归档查看和导出审计日志。

## 电子会计档案 Phase 7 边界

- 支持从凭证附件生成 `ArchiveDocument`，以 `source_type/source_id` 关联凭证、账套和期间。
- 支持计算并保存上传字节的 `sha256_hash`，默认只保存元数据和可选文本摘录，不把原始二进制永久写入 SQLite。
- 支持 `ocr_status` 与 `verification_status` 分离：文本文件标记为 `text_parsed`，图片/PDF 在缺少 OCR 引擎时标记为 `engine_required`；发票和银行回单在外部验真未接入时标记为 `pending_external`。
- 支持按账套、期间和案卷类型创建档案案卷，并下载包含 `manifest.json`、文档元数据 JSON 和文本摘录的 ZIP 归档包。
- 支持 `archive.read`、`archive.document.create`、`archive.case.create`、`archive.package.download`、`archive.verification.update` 权限点和 `archive.*` 审计事件。
- 当前不提供 CA 签章、官方电子发票验真、对象存储 WORM、长期冷备、电子会计档案国标全量字段、跨系统移交、档案销毁审批或原始文件永久存储。
