# 企业级体验收敛审计笔记

审计时间：2026-07-08

审计对象：已安装桌面版 `ChinaFinanceAIAssistant.exe`，本地地址 `http://127.0.0.1:8000`。

## 截图证据

- `01-home-dashboard-viewport.png`：首页 Dashboard 首屏。
- `02-data-center-viewport.png`：点击“数据分析”后的视口状态。
- `03-company-settings-viewport.png`：点击“企业设置”后的视口状态。

## 关键发现

1. 首页信息密度偏高，但主线仍清楚。
   - 首屏能看到风险、收入、成本、利润、现金流等核心指标。
   - 问题是顶部说明、指标卡、图表和操作按钮都在抢注意力，缺少“今天先做什么”的任务入口。

2. 导航与页面落点不稳定。
   - 点击“数据分析”后，视口落在报表内容附近，而不是数据运营中心的顶部。
   - 点击“企业设置”后，视口落在期间结账区域，而不是企业设置区域。
   - 这会让用户误以为菜单失效，或认为页面内容混在一起。

3. 模块页重复展示摘要卡、说明文字和动作按钮。
   - 许多模块都使用相似的标题、摘要、状态队列和卡片布局。
   - 对开发演示有帮助，但对真实财务人员来说会增加扫视成本。

4. 功能成熟度没有被明确区分。
   - 已可用、演示数据、待接入、待配置的能力在视觉上很接近。
   - 用户容易把规划态功能当成可直接使用的正式功能。

5. 长滚动页面承担过多职责。
   - 数据中心聚合凭证、账簿、往来、银行、资产、工资、月结、归档等多个工作流。
   - 用户缺少“只看待办”“只看本模块”“进入完整列表”的分层路径。

## 改进方向

第一轮改版应先做产品化收敛，不继续堆叠新模块：

- 首页改为“今日财务任务台”。
- 左侧导航改成少量一级工作区，并支持清晰选中状态。
- 每个模块顶部只保留必要标题、状态标签和主要动作。
- 将详细说明移到帮助、空状态或二级说明中。
- 明确显示功能成熟度：正式可用、演示数据、待配置、待接入。
- 修复锚点导航和滚动落点，避免点击菜单后落到错误区域。
