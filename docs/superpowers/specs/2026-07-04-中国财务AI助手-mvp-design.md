# China Finance AI Assistant MVP 设计规格

## 1. 项目定位

项目名称：**China Finance AI Assistant**

第一阶段产品定位：面向中国企业财务经理的经营分析与风险预警驾驶舱。系统不是聊天机器人，而是一套辅助财务人员分析经营数据、识别异常、生成管理报告的智能财务工作台。

长期愿景：逐步演进为 **中国企业智能财务操作系统（China Intelligent Finance OS）**，覆盖财务工作层、税务智能层、数据决策层和多 Agent 协同层。

第一版 MVP 不追求全量财税自动化，先验证一个核心闭环：

```text
内置示例财务数据
  -> 后端生成经营指标、趋势、风险提示、图表数据
  -> 前端展示总览式驾驶舱
  -> AI 生成经营分析摘要和管理建议
  -> 用户查看风险明细与报告草稿
```

## 2. 第一版范围

第一版明确实现：

- 示例企业 12 个月财务数据。
- 经营总览驾驶舱。
- 收入、成本、利润趋势分析。
- 费用结构分析。
- 利润瀑布分析。
- 现金流分析。
- 风险等级与异常清单。
- 风险热力图。
- AI 经营摘要。
- 经营分析报告草稿。

第一版明确不实现：

- 真实 Excel 上传。
- OCR 票据识别。
- 自动记账。
- 自动生成会计凭证。
- 自动审核凭证。
- 自动申报税务。
- 每日政策同步。
- 真实大模型联网法规检索。
- 企业级权限、多租户和组织架构。

这些能力进入后续路线，但第一版需要预留清晰接口边界。

## 3. 技术路线

第一版采用：

```text
Python FastAPI + React + ECharts
```

选择原因：

- Python 更适合后续 Excel、PDF、OCR、财务数据分析、政策抓取和批处理任务。
- FastAPI 适合快速构建结构清晰、可测试的 API。
- React 与 ECharts 适合构建管理驾驶舱、趋势图、热力图、瀑布图和现金流图。
- 第一版报告先由规则模板生成，后续可在接口不变的前提下替换为大模型生成。

## 4. 系统分层

```text
React 前端驾驶舱
  -> FastAPI 应用服务
  -> 财务分析服务
  -> 风险预警服务
  -> 报告生成服务
  -> 示例数据与指标模型
```

模块职责：

- React 前端驾驶舱：展示经营总览、图表、风险预警、AI 摘要和报告草稿。
- FastAPI 应用服务：提供健康检查、驾驶舱、风险、报告和示例数据接口。
- 财务分析服务：计算收入、利润、费用率、现金流、同比、环比等指标。
- 风险预警服务：识别费用率异常、现金流与利润背离、库存周转异常、税负率异常等风险。
- 报告生成服务：根据指标和风险项生成经营分析报告草稿。
- 示例数据与指标模型：提供稳定、可复现的 12 个月演示数据。

## 5. 页面与交互

第一版不做营销首页。用户打开系统后直接进入经营分析驾驶舱。

主布局采用“总览式驾驶舱”为主，同时加入“风控优先驾驶舱”的风险模块：

```text
顶部栏：
项目名、示例企业、期间选择、生成报告按钮

左侧导航：
经营总览
利润分析
现金流分析
风险预警
AI 报告

主驾驶舱：
核心指标卡片
收入/成本/利润趋势图
费用结构图
现金流趋势图
风险预警模块
风险热力图
AI 经营摘要
```

交互流程：

```text
1. 用户打开系统，默认加载示例企业 12 个月数据。
2. 用户切换月份或季度，图表和指标同步刷新。
3. 用户点击风险项，查看异常描述、风险等级、触发原因、建议检查资料和谨慎合规提示。
4. 用户点击“生成报告”，系统生成经营分析报告草稿。
5. 用户在 AI 报告页查看利润分析、资金分析、风险提示和管理建议。
```

前端风格：专业、克制、管理驾驶舱风格。AI 能力内嵌在分析、风险和报告中，不以聊天窗口作为主界面。

- 收入环比。
- 毛利率。
- 净利率。

- 现金流利润比。
- 应收账款周转天数。

- 现金流与利润背离。
- 应收账款回款异常。
- 库存周转异常。
- 税负率异常。

图表指标：

- 费用结构。
- 现金流趋势。
- 风险热力图。
- 利润瀑布图。

## 8. 风险等级与合规措辞

风险等级统一使用五星表达：

```text
★☆☆☆☆ 低风险
★★☆☆☆ 关注
★★★☆☆ 需复核
★★★★☆ 高风险
★★★★★ 严重风险
```

第一版所有税务相关内容只做经营与财税风险提示，不下“违法”“违规已成立”或“确定存在税务问题”等结论。

涉及税务、政策、金税四期时，统一使用谨慎表述：

```text
建议财务人员结合企业实际业务、申报口径和最新政策进一步复核。
```

涉及中国会计准则、税法、财政部规定、国家税务总局公告时，系统不得编造法规。第一版没有接入真实法规库，因此报告中不生成具体法规引用；后续接入法规库后，必须注明法规名称、发布机关、发布日期、失效状态和更新时间。

## 9. 后端 API

第一版 API 草案：

```text
GET /health
返回服务状态。

GET /api/v1/dashboard/overview?period=2026-06
返回核心指标、图表数据、AI 摘要、风险摘要。

GET /api/v1/dashboard/risks?period=2026-06
返回风险项列表、风险等级、触发原因、建议检查资料。

GET /api/v1/dashboard/report?period=2026-06
返回经营分析报告草稿。

GET /api/v1/dashboard/sample-data
返回示例数据，用于调试和后续 Excel 模板对照。
```

错误处理规则：

- `period` 不存在：返回 `404`，并提示可用期间范围。
- `period` 格式错误：返回 `422`，要求格式为 `YYYY-MM`。
- 服务内部计算错误：返回 `500`，不暴露内部堆栈。

## 11. 后续路线

第二阶段：接入标准 Excel 模板上传，支持利润表、资产负债表、现金流量表解析。

第三阶段：接入政策法规库，覆盖国家税务总局、财政部、国务院、海关总署、人民银行、证监会等来源，记录新增、修改、废止和版本状态。

第四阶段：建立税务 Agent、会计 Agent、预算 Agent、经营 Agent、风险控制 Agent 和总调度 Agent。

第五阶段：扩展 OCR、自动记账、凭证生成、AI 审计、RPA、电子税务局辅助和金税风险分析。

## 12. 验收标准

第一版设计完成后的实现应满足：

- 打开系统直接看到经营分析驾驶舱。
- 示例数据可驱动所有核心指标和图表。
- 风险模块能展示等级、触发原因和建议检查资料。
- 报告草稿包含利润分析、资金分析、风险提示和管理建议。
- 所有税务相关措辞保持谨慎，不编造法规引用。
- 后端指标和风险规则有自动化测试覆盖。
- 前端桌面和移动视口无明显遮挡、溢出或空白图表。
